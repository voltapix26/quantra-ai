Quantra AI — Notes

What this is, how it's built, and exactly how to run it. Everything you need is on this page.

i What this is

A live market-analysis terminal for crypto and stocks. Pick any asset and Quantra pulls real price history, computes technical + fundamental indicators, writes a plain-language AI verdict, and projects a calibrated probabilistic forecast. You can export the analysis to Excel and a PDF report, and read the latest news for any stock.

Crypto: CoinGecko Stocks & News: Yahoo Finance API keys: none Dependencies: none (Node built-ins) Default port: 5280

Analysis only — no automatic orders. Not investment advice.

▶ How to run

You only need Node.js 18+ installed. No npm install — the server uses only built-in modules.

Windows · PowerShell

# 1. go to the project folder
cd C:\Users\eshan\quantra-terminal

# 2. start the server
node server.js

# 3. open in your browser
#    http://localhost:5280

macOS / Linux · Terminal

cd quantra-terminal
node server.js
# then open http://localhost:5280

Open it

  • Terminal: http://localhost:5280
  • News: http://localhost:5280/news.html
  • These notes: http://localhost:5280/notes.html

Use a different port

# PowerShell
$env:PORT=8080; node server.js

# macOS / Linux
PORT=8080 node server.js

Stop the server

Press Ctrl + C in the terminal window running it.

Optional: AI-written verdicts

The core app needs no API key. To have the Quantra AI reasoning engine rewrite each verdict in richer language and return a comprehension-based news-impact score, set an LLM API key (ANTHROPIC_API_KEY) before starting the server (the SDK is already installed):

# PowerShell
$env:ANTHROPIC_API_KEY='sk-ant-...'
node server.js

# macOS / Linux
export ANTHROPIC_API_KEY='sk-ant-...'
node server.js

The verdict badge shows ✦ Quantra AI when active, or rule-based when no key is set. Override the model with QUANTRA_AI_MODEL.

Note: open the app through http://localhost:5280, not by double-clicking index.html. Opening the file directly (file://) works for crypto only — stocks, fundamentals and news need the local server because Yahoo Finance can't be called straight from the browser.

★ What you can do

  • Analyse any asset — switch Crypto/Stocks, or search any symbol (incl. Indian tickers like RELIANCE.NS).
  • Deep technical suite — SMA-20/50/200, RSI, MACD, Bollinger, ATR, Stochastic, ADX and OBV/volume, shown as a transparent bull/bear signal breakdown.
  • Walk-forward accuracy — Quantra learns per-asset signal weights on 70% of history and validates on the unseen 30%, reporting the out-of-sample hit-rate so the confidence figure is grounded, not guessed.
  • Technical + fundamental verdict — the above blended with screener-style ratios into one read.
  • Analyst estimates & peers — consensus, price-target range, forward EPS/revenue estimates, next earnings date, and a one-click peer comparison.
  • AI-written verdict (optional) — with an API key set, the verdict is rewritten by the Quantra AI reasoning engine from the computed indicators + fundamentals + news, and returns a comprehension-based news-impact score that sharpens the forecast; without one, it falls back to the built-in rule-based read. A badge shows which is active.
  • Pick range & interval — 1M–2Y, daily or weekly; optionally add a Compare symbol.
  • ⬇ Excel — workbook with Summary, full price/indicator History (with the compare column), and the Forecast table.
  • 📄 PDF report — verdict, price+forecast chart, the future prediction, and fundamentals.
  • 📰 News — latest headlines for the selected stock.

? Troubleshooting

  • "node is not recognized" — install Node.js 18+ from nodejs.org, then reopen the terminal.
  • Stocks/news won't load — make sure you opened localhost:5280 (not the file), and the server window is still running.
  • Port already in use — start on another port: $env:PORT=8080; node server.js.
  • Fundamentals empty — Yahoo rate-limits; wait a few seconds and reselect. Crypto has no fundamentals by design.
  • Excel/PDF button does nothing — those use CDN libraries; if you're offline, Excel falls back to CSV. Reconnect for full .xlsx/PDF.
